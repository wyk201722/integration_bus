## SLO caveats (carried forward as follow-ups)

These do NOT block this milestone — they are documented in the signoff and registered as items for the next phase:

1. **Loopback latency only** — SLO numbers measured on the same host with no network hop. Production over LAN/container will be higher; the 180–595× headroom is comfortable but not equivalent to an in-environment bench.
2. **Search empty-hits fast path** — `healthz` reports `qdrant_ok=false` and `kg_store_ok=false` for this session, so `/v1/search` returns `hits=[]` and the 0.84ms P95 measures the empty-result path, not vector retrieval. A re-bench against a populated Qdrant is required to validate the real search SLO.
3. **WS-push P95 is combined REST+push** — 5.56ms bounds the REST write + WS push round-trip together. Splitting for finer accounting is optional.
4. **candidate_signal_ids placeholder ULIDs work because validation is shape-only** — KB confirmed no referential-integrity check (`kb.jsonl` request_help @ 2026-05-15T13:09:11). If the contract later adds FK enforcement, fixtures will need to pre-seed a real event entry.
5. **SLO benchmark integration** — `test_slo_live.py` is on-demand via `make test-slo`; decide later whether to fold into `make ci`.
